<Title>Styling</Title>

`JBButton` styling is variable-driven. Use `--jb-button-*` variables on the host to control size, colors, variants, loading color, disabled behavior, and content spacing.

## Common Design Tokens

Use the core theme docs as the source of truth for shared design-system variables:

- [Theme Colors](../../jb-core/theme/stories/colors.mdx) explains the general color palette variables, default colors, customization approaches, and color generation helpers.
- [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) explains shared size and radius variables.
- [Theme Custom Galleries](https://javadbat.github.io/design-system/?path=/docs/theme-custom-material--docs) gathers the shared custom style recipes across components.

Button recipes should use those common tokens where possible, then map them into `--jb-button-*` variables where the button needs component-specific values.

## Styling Model

Use four layers when styling button examples:

- Common design tokens are documented in [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx).
- Reusable example theme packs live in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) and can be shared across components.
- Component recipe files live in [`modules/jb-button/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-button/stories/styles).
- Public component variables use `--jb-button-*` on `jb-button`.

Each example uses one shared example theme pack from [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) and one button-specific recipe from [`modules/jb-button/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-button/stories/styles). The style story applies both classes directly to `JBButton`.

## Recommended Customization Path

1. Use `--jb-button-*` variables first for size, color, border, radius, shadow, content gap, and loading color.
2. Use `variant`, `color`, and `size` attributes before creating separate CSS classes for common button families.
3. Override variant-specific variables such as `--jb-button-bg-color-outline-hover` only when the general variables are not specific enough.
4. Use `jb-button:state(disabled)` for custom disabled treatment that cannot be expressed with disabled variables.
5. Use common theme variables from [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) when the style should align with the design system.
6. Put reusable theme-pack values in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) only when they should be shared by multiple component demos.
7. Add a new button variable or official part only when a needed internal surface cannot be customized with current public variables.

## Starter Recipe

Use this minimal recipe when a product only needs brand alignment across button variants.

```css
jb-button.brand-button {
  --jb-button-primary: #2563eb;
  --jb-button-secondary: #475569;
  --jb-button-danger: #dc2626;
  --jb-button-border-radius: 0.75rem;
  --jb-button-font-weight: 700;
  --jb-button-content-gap: 0.5rem;
  --jb-button-bg-color-hover: #1d4ed8;
  --jb-button-bg-color-pressed: #1e40af;
  --jb-button-box-shadow: 0 0.75rem 1.5rem rgb(37 99 235 / 0.18);
  --jb-button-bg-color-outline: #ffffff;
  --jb-button-bg-color-outline-hover: #eff6ff;
  --jb-button-bg-color-ghost-hover: #eff6ff;
  --jb-button-bg-color-text-hover: #eff6ff;
}

jb-button.brand-button:state(disabled) {
  --jb-button-box-shadow: none;
}
```

## Theme File Pattern

Use [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) as the common design-token guides. Theme-pack files should keep reusable demo variables in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles), then each component should keep its own style CSS in its story [`styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-button/stories/styles) folder.

```css
/* https://github.com/javadbat/design-system/blob/main/docs/styles/carbon.css */
.theme-name-style {
  /* shared design-system variables */
}

/* https://github.com/javadbat/design-system/blob/main/modules/jb-button/stories/styles/style-carbon.css */
jb-button.theme-name-button {
  /* button host variables */
}

jb-button.theme-name-button:state(disabled) {
  /* disabled-only refinements when variables are not enough */
}
```

This keeps common theme tokens reusable while making the `JBButton` styling API visible in one page.
